# corparius, product context

## What it is

corparius runs autonomous AI micro-companies on the operator's own machine. The web console (app/webui.py) is the operator's control surface: watch a company work, unblock what waits for a human, steer the backlog, wire LLM providers and talk to the CEO agent.

## Users

Self-hosters and homelab operators. They run the stack on their own hardware, check in once or twice a day, and value auditability over convenience. They read logs. They distrust magic.

## Register

product. The console serves the task; it should disappear into it. Earned familiarity over novelty.

## Tone

Calm, precise, factual. Labels state what things are. No exclamation marks, no praise, no marketing adjectives. Empty states teach the next command instead of apologizing.

## Anti-references

Hosted SaaS dashboards that hide state behind onboarding tours; crypto-trader dashboards (dense neon, blinking numbers); anything requiring a login page for localhost.

## Strategic principles

- Local first: every screen must be honest about mock mode and offline state.
- The human gate is the point: approvals are the primary action, never buried.
- One accent, used for action and health, never decoration.
